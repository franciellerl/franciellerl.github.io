// visão_geral // overview

Pipeline completo de engenharia de dados aplicado a dados de biodiversidade floral brasileira. A fonte é o GBIF (Global Biodiversity Information Facility), banco de dados global e gratuito com mais de 2 bilhões de registros de ocorrencia de espécies. O projeto cobre todo o ciclo: coleta via API, limpeza, análise exploratória, clustering geoespacial e mapa interativo como entrega final.

End-to-end data engineering pipeline applied to Brazilian floral biodiversity data. The data source is GBIF (Global Biodiversity Information Facility), a free global database with over 2 billion species occurrence records. The project covers the full cycle: API collection, cleaning, exploratory analysis, geospatial clustering, and an interactive map as the final deliverable.

→ 9.000 registros coletados via API REST paginada
→ Pipeline ETL com limpeza e validação
→ Clustering K-Means com K ideal
→ Mapa interativo com heatmap, marcadores e toggle por cluster → 9,000 records collected via paginated REST API
→ ETL pipeline with cleaning and validation
→ K-Means clustering with optimal K
→ Interactive map with heatmap, markers and per-cluster toggle

// estrutura_do_pipeline // pipeline_structure

01
pipeline.ipynb — Coleta e Limpeza Collection and Cleaning

Requisições paginadas à API REST do GBIF (300 registros/página) com tratamento de erros e pausa entre chamadas. Seleção de colunas, remoção de nulos, validação de coordenadas pelo bounding box e pela malha geográfica do Brasil, normalização de texto e deduplicação por gbifID. Saída em .parquet e .csv.

Paginated requests to GBIF REST API (300 records/page) with error handling and rate-limiting pauses. Column selection, null removal, coordinate validation against Brazil's bounding box and national boundary geometry, text normalization and deduplication by gbifID. Output in .parquet (~5x smaller than CSV) and .csv.

02
analise.ipynb — EDA + Clustering K-Means EDA + K-Means Clustering

Análise exploratória por meio de três gráficos: Top 10 espécies, Top 10 estados e distribuição das fontes. Clustering K-Means sobre latitude e longitude normalizadas; K ideal escolhido pelo equilíbrio entre os testes (testado de K=2 a K=8).

Exploratory analysis across three charts:Top 10 species, Top 10 states, and source distribution. K-Means clustering on normalized latitude and longitude; optimal K was determined by balancing metrics, (tested from K=2 to K=8).

03
geovisualizacao.ipynb — Visualização com Folium Visualization with Folium

Mapa construído com Folium (wrapper Python para Leaflet.js). Camadas incluem heatmap de densidade com todos os registros, 2.000 marcadores interativos com popup por espécie, e uma camada separada por cluster com toggle individual. Saída como mapa_plantas_brasil.html

Map built with Folium (Python wrapper for Leaflet.js). Layers include density heatmap with all records, 2,000 interactive markers with per-species popup, and a separate layer per cluster with individual toggle. Output as mapa_plantas_brasil.html
